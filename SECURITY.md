# Security Policy

## Supported Version

Security fixes will be made against the current `main` branch on a best-effort basis.

## Reporting a Vulnerability

Do not open a public GitHub issue for a suspected vulnerability.

Use one of these paths instead:

1. GitHub private vulnerability reporting for this repository, if it is enabled.
2. If private reporting is not available, contact the maintainer through the repository owner profile: `https://github.com/cotopena`

When reporting, include:

- affected area or file path
- impact summary
- reproduction steps or proof of concept
- any suggested mitigation if you already have one

## Disclosure Expectations

- I will try to acknowledge valid reports within 7 days.
- I will try to keep reporters informed about fix status and disclosure timing.
- Please avoid public disclosure until a fix or mitigation is available.
